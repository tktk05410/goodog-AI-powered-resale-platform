# GoodDog 校园二手交易平台 — 项目结构设计文档

## 一、项目概述

GoodDog 是一个面向校园场景的二手交易平台，支持商品发布（出售/求购）、即时通讯、交易管理、数据统计、管理员后台等功能。系统集成 Qwen3.7-plus 多模态 AI，实现商品自动识别与标签生成。

**技术栈总览：**

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3 + Vite + Element Plus + ECharts + Pinia + Vue Router |
| 后端 | Flask + Flask-SQLAlchemy + PyJWT |
| AI 服务 | Qwen3.7-plus（文本分类 + 图像识别） |
| 数据库 | MySQL 8.0 |
| 部署 | Docker + Nginx |

## 四、后端架构设计

### 4.1 应用工厂模式

后端采用 Flask 应用工厂模式，通过 `create_app()` 函数创建应用实例，支持多环境配置切换。

```
create_app(config_name)
    ├── 加载配置 (config.py)
    ├── 初始化数据库 (db.init_app)
    ├── 注册蓝图 (register_blueprints)
    ├── 注册钩子 (register_hooks)
    │     ├── before_request → JWT 解析、用户身份识别
    │     └── after_request  → 响应时间统计 (X-Response-Time)
    └── 注册错误处理 (register_error_handlers)
          ├── 400 Bad Request
          ├── 401 Unauthorized
          ├── 403 Forbidden
          ├── 404 Not Found
          └── 500 Internal Server Error
```

### 4.2 配置管理

`config.py` 采用类继承结构管理多环境配置：

| 配置类 | 用途 | 关键差异 |
|--------|------|----------|
| `Config` | 基础配置 | 数据库连接、JWT密钥、上传配置 |
| `DevelopmentConfig` | 开发环境 | DEBUG=True, SQLALCHEMY_ECHO=True |
| `ProductionConfig` | 生产环境 | DEBUG=False, SQLALCHEMY_ECHO=False |

### 4.3 API 路由设计

所有 API 统一使用 `/api` 前缀，通过 Flask Blueprint 模块化组织：

| 模块 | URL 前缀 | 功能说明 | 认证要求 |
|------|----------|----------|----------|
| auth | `/api/auth` | 登录、注册、JWT 令牌、修改密码 | 部分需要 |
| products | `/api/products` | 商品 CRUD、我的商品列表 | 写操作需要 |
| transactions | `/api/transactions` | 订单创建、支付、确认、取消 | 需要 |
| messages | `/api/messages` | 消息发送、会话列表、已读标记 | 需要 |
| users | `/api/users` | 个人信息、信用分、排行榜 | 部分需要 |
| ai | `/api/ai` | 文本分类、图像识别、价格估算、文案生成 | 需要 |
| logs | `/api/logs` | 系统日志查询 | 需要 |
| stats | `/api/stats` | 数据概览、分布统计、趋势分析、活跃时段 | 需要 |
| admin | `/api/admin` | 数据统计、用户列表、商品列表、系统日志 | 管理员 |
| tags | `/api/tags` | 标签 CRUD、商品标签管理 | 写操作需要 |

### 4.4 认证机制

采用 JWT（JSON Web Token）无状态认证：

```
客户端请求 → Authorization: Bearer <token>
                ↓
        before_request 钩子
                ↓
        JWT 解码 → 获取 user_id
                ↓
        查询 User 模型 → 挂载到 g.user
                ↓
        token_required 装饰器 → 校验 g.user 是否存在
```

- Token 有效期：86400 秒（24 小时）
- 密码存储：bcrypt 哈希加密
- 角色系统：`user` / `admin`，管理员可编辑/删除任意商品

- **tag**: idx_name

## 六、AI 服务设计

### 6.1 架构

AI 服务独立于后端路由，封装在 `ai_services/` 目录中，通过路由层暴露 API 接口。

```
ai_services/
├── text_classifier.py
│   ├── TextClassifier          # 文本分类与标签生成
│   │   ├── generate_tags_with_qwen()      # Qwen API 生成标签
│   │   ├── estimate_price_with_qwen()     # Qwen API 估算价格
│   │   ├── generate_copywriting()         # AI 文案生成
│   │   └── generate_tags_by_keywords()    # 关键词回退方案
│   ├── ContentFilter           # 敏感内容过滤
│   │   ├── contains_sensitive()
│   │   └── filter_text()
│   └── AIMessageSummarizer     # 消息摘要
│       └── summarize()
│
└── image_processor.py
    └── ImageProcessor          # 图像处理
        ├── read_image()                    # 图像读取
        ├── to_grayscale()                  # 灰度化
        ├── denoise()                       # 去噪
        ├── detect_edges()                  # 边缘检测
        ├── resize_image()                  # 缩放
        ├── extract_histogram()             # 直方图提取
        ├── extract_features()              # 特征提取
        ├── compare_histograms()            # 直方图对比
        ├── image_to_base64()               # Base64 编码
        └── recognize_with_qwen()           # Qwen 多模态识别
```

### 6.2 AI 调用流程

```
发布商品 → 提交标题 + 描述 + 图片
              │
              ├──→ TextClassifier.generate_tags_with_qwen()
              │       ├── 有 API Key → 调用 Qwen3.7-plus 文本模型
              │       └── 无 API Key → 关键词匹配回退
              │
              ├──→ ImageProcessor.recognize_with_qwen()
              │       ├── 图像缩放 → Base64 编码
              │       ├── 调用 Qwen3.7-plus 多模态模型
              │       └── 返回图片标签
              │
              ├──→ TextClassifier.estimate_price_with_qwen()
              │       ├── 调用 Qwen3.7-plus 估算价格区间
              │       └── 返回 {min_price, max_price}
              │
              └──→ TextClassifier.generate_copywriting()
                      ├── 调用 Qwen3.7-plus 生成商品文案
                      └── 返回优化后的商品描述
```

### 6.3 降级策略

- Qwen API 不可用时，文本标签生成自动降级为关键词匹配方案
- 图像识别失败时返回默认标签 `['二手商品']`
- 价格估算失败时返回 `{min_price: 0, max_price: 0}`
- 文案生成失败时返回错误提示，不影响主流程

## 九、核心业务流程

### 9.1 商品发布流程

```
用户填写商品信息（标题、描述、价格、图片）
        │
        ▼
前端 FormData 提交 → POST /api/products/
        │
        ▼
后端接收 → 保存图片到 uploads/
        │
        ├──→ AI 文本标签生成（TextClassifier）
        ├──→ AI 图像标签生成（ImageProcessor）
        ├──→ AI 价格估算（TextClassifier）
        └──→ AI 文案生成（TextClassifier）
        │
        ▼
保存商品记录 → 保存 AI 生成的标签（product_tag）
        │
        ▼
返回商品信息（含标签）
```

### 9.2 交易状态流转

```
创建订单(pending) → 支付(paid) → 确认完成(done)
        │                │
        └──→ 取消(canceled) ←┘
```

### 9.3 消息通讯流程

```
发送方 → POST /api/messages/ → 保存消息记录
                              → AI 生成消息摘要（AIMessageSummarizer）
        │
        ▼
接收方 → GET /api/messages/?user_id=xxx → 获取会话消息
       → PUT /api/messages/:id/read     → 标记已读
       → GET /api/messages/unread-count → 获取未读数
```

### 9.4 管理员后台流程

```
管理员登录 → role='admin' 标识
        │
        ▼
导航栏显示"管理"入口
        │
        ▼
管理后台功能：
    ├── 数据概览（注册用户、商品总数、在售/已售、消息总数）
    ├── 用户管理（查看用户列表、删除非管理员账号）
    ├── 商品管理（查看所有商品、直接编辑/删除）
    ├── 系统日志（查看系统操作记录）
    └── 消息中心（与普通用户相同的聊天界面）
```

## 十、安全设计

| 安全措施 | 实现方式 |
|----------|----------|
| 密码安全 | bcrypt 哈希存储，不存储明文 |
| 身份认证 | JWT Token + 请求头 Bearer 认证 |
| 角色权限 | `user` / `admin` 角色，管理员可编辑/删除任意商品 |
| 路由守卫 | 前端 Vue Router 拦截 + 后端 token_required 装饰器 |
